# 基本知识

1. 标签按关系可分为：“包含关系”和“并列关系”。
	```html
	<!--包含关系-->
		<head>
			<title> </title>
		</head>
	``` 
```html
<!--并列关系-->
	<head> </head>
	<body> </body>
```

2. 标签按个数可分为：”双标签“和”单标签“
	```html
		<!--双标签-->
		<html> </html>
		
		<!--单标签 -->
		<br/>
	```

## 根标签

html文件的骨架，页面内容也都是在这些基本标签上书写。
如果你装了xx插件，可直接输入[!+Tab]键来快速生成骨架。
```html
	<html> --- <!--页面中最大的标签，我们叫它“根标签”-->
		<head> --- <!--文档的头部-->
			<title>一个页面</title> --- <!--文档的标题，就是网页标签上的名字-->
		</head>
		<body> --- <!--文档的主体，页面内容基本都是放到里面的-->
			主体内容
		</body>
	</html>
```

## 文档类型声明标签
```html
	<!DOCTYPE> --- <!--告诉浏览器使用哪个HTML版本来显示网页->
	<!DOCTYPE html> --- <!--
							1.这里表示使用HTML5。
							2.位置必须声明在文档最前面。
							3.该标签不是一个HTNL标签，它是一个文档类型 声明标签。
							4.该标签不区分大小写。 
						-->
```

lang属性用来定义当前文档显示语言。
```html
		<html lang="en"> --- "n"表示英语
							 "zh-CN"表示中文
```

<meta>标签
**charset**属性用来规定文档使用的字符编码。
注意在编写时统一编码方式，防止出现乱码。
```html
	<meta charset="UTF-8"> --- 规定文档使用utf-8编码
```

# HTML标题标签\<h1-h6>
一共6个等级。
```html
    <h1>这是一级标题</h1> 
    <h2>这是二级标题</h2>
    <h3>这是三级标题</h3>
    <h4>这是四级标题</h4>
    <h5>这是五级标题</h5>
    <h6>这是六级标题</h6>
```
特点：
1. 加了标题的文字会变的加粗，字号也会依次变大.
2. 标签从大到小，字号依次减小。
3. 一个标题独占一行。

注意：
1. \<h1>标签只使用一次。
2. 级别越大的标签，其强调性越重，故不要乱用标签。

## 路径

**目录文件夹**：就是普通文件夹，里面只不过存放了我们做页面所需要的相关素材，比如 html 文件、图片等。
**根目录**：打开目录文件夹的第一层就是根目录

**相对路径**：以所在html文件的存储路径为准，从当前位置出发到最终位置
		存放在同级目录：文件名
        存放在下级目录：./文件夹名/文件名 
        存放在上级目录：../文件夹名/文件名
        
绝对路径：从盘符出发到最终位置，如：C：/文件夹名/文件名

在 HTML 标签中，\<a> 标签用于定义超链接，作用是从一个页面链接到另一个页面。

```html

```

注意：
1. \<a href="#"><\/a> 表示空链接。
2. 超链接可以与img标签一起使用，达到点击图片跳转网页的效果

# HTML列表标签
表格是用来显示数据的，那么列表就是用来布局的。
列表最大的特点就是**整齐**、**整洁**、**有序**，它作为布局会更加自由和方便。
根据使用情景不同，列表可以分为三大类：**无序列表**、**有序列表**和**自定义列表**。

## 1.\<ul>无序列表
使用场景：在网页中表示一组无顺序之分的列表，如：新闻列表，最常用。
```html
ul>li>任意标签
<ul>
    <li>列表1</li>
    <li>列表2</li>
    <li><img src="URL" alt=""></li> 在列表里显示图片
</ul>
```

| 标签名 | 说明                     |
| :-: | ---------------------- |
| ul  | 表示无序列表的整体，用于包裹li标签。    |
| li  | 表示无序列表的每一项，用于包含每一行的内容。 |
显示特点：
	“•“ 列表的每一项前默认显示圆点标识。

注意点：
1. ul标签中只允许包含li标签。
2. li标签可以包含任意内容。
3. 无序列表会带有自己的样式属性，但在实际使用时，我们会使用 CSS 来设置。
4. 无序列表的各个列表项之间没有顺序级别之分，是并列的。

## 2.\<ol>有序列表
使用场景：在网页中表示一组有顺序之分的列表，如：排行榜，偶尔用。
```html
ol>li>任意标签
<oi>
    <li>有序列表1</li>
    <li>有序列表2</li>
    <li><a href="URL" target="__blank"> </a></li> 可以在列表里添加超链接
</oi>
```

^89957a

| 标签名 | 说明                     |
| :-: | ---------------------- |
| ol  | 表示有序列表的整体，用于包裹标签。      |
| li  | 表示有序列表的每一项，用于包含每一行的内容。 |
显示特点：
	列表的每一项前默认显示序号标识。

注意点：
1. ol标签中只允许包含li标签。
2. li标签可以包含任意内容。
3. 有序列表会带有自己样式属性，但在实际使用时，我们会使用 CSS 来设置。

## 3.\<dl>自定义列表
自定义列表常用于对术语或名词进行解释和描述，定义列表的列表项前没有任何项目符号。

使用场景：在网页的底部导航中通常会使用自定义列表实现，常用作制作**底部导航栏**。

```html
dl>dt或dd
<dl>
    <dt>表头</dt>
    <dd>自定义列表1</dd>
    <dd>自定义列表2</dd>
</dl>
```

| 标签名 | 说明                      |
| :-: | ----------------------- |
| dl  | 表示自定义列表的整体，用于包裹dt/dd标签。 |
| dt  | 表示自定义列表的主题。             |
| dd  | 表示自定义列表的针对主题的每一项内容。     |
显示特点：
	dd前会默认显示缩进效果。

注意点：
1. dl标签中**只允许**包含dt/dd标签。
2. dt/dd标签可以包含**任意内容**。

## 3. 表格标题和表头单元格标签

| 标签名     | 名称    | 说明                                   |
| ------- | ----- | ------------------------------------ |

|  标签名  |  名称  |
| :---: | :--: |
| thead | 表格头部 |
| tbody | 表格主体 |
| tfoot | 表格底部 |
![[assets/图片/HTML/表格的结构标签.png]]
注意点：
1. 表格结构标签写在\<table>标签内部
2. 表格结构标签内部用于包裹tr标签。
3. 表格的结构标签可以省略。

## 5. 合并单元格属性
使用场景：将水平或垂直多个单元格合并成一个单元格。

遵循**左上原则**，确定保留谁删除谁：
1. 上下合并→只保留最上的，删除其他
2. 左右合并→只保留最左的，删除其他

| 属性名     | 属性值      | 说明                    |
| ------- | -------- | --------------------- |
| rowspan | 合并单元格的个数 | 跨行合并，将多行的单元格**垂直合并**。 |
| colspan | 合并单元格的个数 | 跨列合并，将多列的单元格**水平合并**。 |
注意点:
1. 只有同一个结构标签中的单元格才能合并，**不能跨结构标签合并**（不能跨：thead、tbody、tfoot）
2.    合并单元格属性写在哪个\<td>标签内，这个标签就是合并的原点，且遵守左上原则。

```html
合并单元格，左上原则，被合并的单元格记得要删除
在td内：
	<td rowspan="跨行合并个数"> </td>
	<td colspan="跨列合并个数"> </td>
```

# HTML表单域标签\<form>

表单域是一个包含表单元素的区域。
在 HTML 标签中，\<form> 标签用于定义表单域，以实现用户信息的**收集和传递**。
**\<form> 会把它范围内的表单元素信息提交给服务器**。
```html
创建表单，需要创建表单域form
用于承载其余表单内容
<form action="URL地址" method="提交方式" name="表单域的名称">
	各种表单元素控件。
</form>
```

| 属性     | 属性值      | 作用                          |
| ------ | -------- | --------------------------- |
| action | URL地址    | 用于指定接收并处理表单数据的服务器程序的url地址。  |
| method | get/post | 用于设置表单数据的提交方式，其取值为get或post. |
| name   | 名称       | 用于指定表单的名称，以区分同一个页面中的多个表单域。  |

# HMTL链接标签\<link>
`<link>`标签是HTML中用于链接外部资源到当前文档的标签，常用于**链接CSS样式表**。它是一个空元素（没有闭合标签），必须放在文档的`<head>`部分。
基本语法：
```html
<link rel="stylesheet" type="text/css" href="样式表路径">
```

| 常用属性     | 属性内容                                                       | 说明              |
| -------- | ---------------------------------------------------------- | --------------- |
| rel(必需)  | 1. stylesheet - 链接CSS样式表。<br>2. icon - 链接网站图标。             | 定义当前文档与链接资源的关系。 |
| href(必需) | URL                                                        | 指定外部资源的路径。      |
| type     | 对于CSS文件，通常是text/css                                        | 指定链接资源的MIME类型。  |
| media    | 1. screen - 计算机屏幕。<br>2. print - 打印预览模式。<br>3. all - 所有设备。 | 指定样式表适用的媒体类型。   |

举个例子：
```css
div{background-color: rgb(0, 255, 38);}
```
```html
<head>
	<link rel="stylesheet" href="验证.css">
</head>

<body>
    <div>
        <p>这是link链接</p>
    </div>
</body>
```
运行结果：\<p>段落变绿。

**优点：**
- **代码分离**：HTML和CSS分开，便于维护。
- **可重用性**：同一CSS文件可用于多个页面。
- **缓存优势**：浏览器可缓存CSS文件，提高加载速度。
- **团队协作**：设计师和开发者可并行工作。
